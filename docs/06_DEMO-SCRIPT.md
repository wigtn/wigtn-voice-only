# Demo Script (2분)

> WIGVO — 부동산 매물 확인 시연

## Pre-Demo Checklist

- [ ] `npm run dev` 실행 중
- [ ] `.env.local` 설정 완료
- [ ] Mock 모드 확인: `ELEVENLABS_MOCK=true` (안전한 기본값)
- [ ] 브라우저 localhost:3000 열어두기
- [ ] 화면 녹화 시작 (백업용)

---

## Demo Flow (2분)

### Scene 1: 문제 소개 (20초)

> "여러분, 직방에서 마음에 드는 방을 찾으셨나요?
> 하지만 '그 방 아직 있나요?' 확인하려면 결국 전화를 해야 합니다.
> 부동산, 수리점, 동네 노포 — 전화가 유일한 관문인 곳이 아직 많습니다.
> WIGVO는 이 전화를 AI가 대신 걸어줍니다."

### Scene 2: 로그인 & 채팅으로 정보 수집 (20초)

**화면**: 로그인 → 채팅 화면 (메인 페이지)

1. (이미 로그인된 상태에서 시작 — 또는 Google 로그인 버튼 클릭 시연)
2. AI 인사: "안녕하세요! 어떤 전화를 대신 걸어드릴까요?"
3. 채팅으로 입력:
   > "직방에서 본 강남역 OO빌라 201호 아직 있는지 확인해줘"
4. AI 응답: "전화번호를 알려주세요!"
5. 채팅으로 입력:
   > "02-1234-5678"

> "Google, Apple, 카카오로 간편하게 로그인하고, AI와 채팅하면서 정보를 알려주면 됩니다."

### Scene 3: 수집 완료 & 전화 시작 (20초)

**화면**: 채팅 화면 하단에 수집 완료 요약 표시

> "AI가 대화에서 필요한 정보를 자동으로 수집합니다."

- AI 요약 메시지:
  - 📍 OO공인중개사 (02-1234-5678)
  - 📋 매물 확인 (OO빌라 201호)
  - "맞으시면 전화 걸어볼게요!"

> "정보가 맞는지 확인하고 [전화 걸기]를 누르면 AI가 실제로 전화를 겁니다."

[📞 전화 걸기] 버튼 클릭

### Scene 4: 통화 중 (20초)

**화면**: 통화 중 페이지 (/calling/[id])

> "지금 AI가 중개사에 전화를 걸고 있습니다.
> '안녕하세요, 앱에서 OO빌라 201호 매물 보고 연락드렸는데요,
> 아직 계약 가능한가요?'
> 이런 식으로 공손하게 대화합니다."

(5초 대기 - mock 모드에서 자동 완료)

### Scene 5: 결과 확인 (20초)

**화면**: 결과 페이지 (/result/[id])

> "결과가 도착했습니다.
> '해당 매물은 계약 가능하며, 오후 6시 방문 가능합니다.'
> AI가 통화 내용을 요약해서 보여줍니다."

- 결과: 매물 확인 완료
- 방문 가능 시간: 오후 6시
- AI 요약 텍스트

### Scene 6: 마무리 (20초)

> "이렇게 전화 한 통 없이 매물 확인이 끝납니다.
> 부동산뿐 아니라 수리 AS 접수, 노포 예약, 병원 문의 등
> 전화가 유일한 관문인 모든 곳에 적용할 수 있습니다.
> 기술 스택은 Next.js 16, ElevenLabs Conversational AI, OpenAI GPT-4입니다.
> 감사합니다."
